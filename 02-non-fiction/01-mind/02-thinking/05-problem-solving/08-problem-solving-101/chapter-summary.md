# Problem Solving 101 — Chapter-by-Chapter Summary

## Reading Guidance

This book is structured as four "classes" (case studies), each introducing problem-solving concepts through a fun scenario. The Tool Box section at the end consolidates all tools. Read the classes first to see the tools in action, then use the Tool Box as reference.

---

## Chapter-by-Chapter Breakdown

### Class #1: Problem-Solving Kid Basics

| Aspect | Detail |
|--------|--------|
| **Core Question** | What is the basic problem-solving process, and how does it work? |
| **What it Covers** | The four-step process; understanding the current situation; setting goals; identifying gaps |
| **Key Concepts** | Problem-solving process, current state, goal state, gap analysis, logic tree introduction |
| **Examples** | A child deciding what to do with their allowance; basic logic tree for personal decisions |
| **Takeaway** | Problem solving starts with clearly understanding where you are and where you want to be. The gap between them is the problem. |
| **Thesis Connection** | Introduces the core four-step model: identify → analyze → solve → implement |

---

### Class #2: Rock Bands and Root Causes

| Aspect | Detail |
|--------|--------|
| **Core Question** | How do you find the real root cause of a problem instead of treating symptoms? |
| **What it Covers** | Root cause analysis; the "why" chain; distinguishing symptoms from causes; using logic trees for diagnosis |
| **Key Concepts** | Root cause, symptom vs. cause, "why" chain, logic tree analysis |
| **Examples** | A rock band with declining concert attendance — is it marketing, quality, or venue? |
| **Takeaway** | Don't treat symptoms. Trace the problem back through "why" questions to find the root cause. Fix that. |
| **Thesis Connection** | Step 2 of the process: analyze root causes using logic trees |

---

### Class #3: Fishy Goals and Solid Achievements

| Aspect | Detail |
|--------|--------|
| **Core Question** | How do you develop a concrete plan to achieve a specific goal? |
| **What it Covers** | Goal setting; creating a problem-solving design plan; hypothesis development; workplan creation |
| **Key Concepts** | Problem-solving design plan, hypothesis, workplan, analysis, solution development |
| **Examples** | An aspiring animator needs to save for a computer — breaking down income and expenses |
| **Takeaway** | A goal without a plan is just a wish. Develop a hypothesis, create a design plan, and execute systematically. |
| **Thesis Connection** | Step 3: develop solutions and create an actionable plan |

---

### Class #4: Soccer School Pros and Cons

| Aspect | Detail |
|--------|--------|
| **Core Question** | How do you make a well-reasoned decision when comparing multiple options? |
| **What it Covers** | Decision-making frameworks; pros and cons; criteria and evaluation; trade-off analysis |
| **Key Concepts** | Pros and cons, criteria, evaluation matrix, trade-offs, objective decision-making |
| **Examples** | A student choosing between soccer schools — comparing facilities, cost, coaching, location |
| **Takeaway** | Use explicit criteria to evaluate options. A decision matrix makes trade-offs visible and decisions defensible. |
| **Thesis Connection** | Step 4 with decision-making emphasis: implement and evaluate using structured tools |

---

### Problem-Solving Tool Boxes

| Tool | Description |
|------|-------------|
| **Logic Tree** | Break down problems into MECE parts for systematic analysis |
| **Yes/No Tree** | Use binary questions to narrow down options or diagnose causes |
| **Problem-Solving Design Plan** | A template that organizes: problem, hypothesis, analysis plan, data needed, timeline |
| **Hypothesis Pyramid** | Structure hypotheses hierarchically from main hypothesis to sub-hypotheses |
| **Pros and Cons; Criteria and Evaluation** | Evaluate options against weighted criteria for objective decision-making |
