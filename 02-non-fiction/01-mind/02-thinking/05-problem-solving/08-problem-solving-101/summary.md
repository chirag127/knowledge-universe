## 💡 The Big Idea in One Paragraph

Problem solving is a skill, not a talent. Anyone can learn it through a simple, four-step approach: (1) understand the current situation, (2) identify the root cause, (3) develop a solution plan, and (4) execute and evaluate. Using a small set of easy-to-learn tools — logic trees, yes/no trees, hypothesis pyramids, pros-and-cons matrices — and a "kid-friendly" case study format (a rock band, a soccer school, a fish pond), Watanabe shows that complex problems can be broken down and solved systematically. Originally written for Japanese schoolchildren, the book's simplicity and clarity made it an international bestseller for all ages.

## 🔑 Thematic Pillars & Core Frameworks

**Pillar 1: The Problem-Solving Process**
Four steps: 1) Identify the problem, 2) Analyze root causes, 3) Develop solutions, 4) Implement and evaluate. The process is iterative, not linear.

**Pillar 2: Logic Trees**
Break complex problems into smaller, manageable components. Two main types: logic trees (for general problem breakdown) and yes/no trees (for decision branching).

**Pillar 3: Hypothesis-Driven Problem Solving**
Start with a hypothesis (an informed guess about the solution), then test it against data. The hypothesis pyramid structures these hypotheses hierarchically.

**Pillar 4: Structured Decision-Making**
Use pros-and-cons matrices and criteria/evaluation frameworks to make decisions transparent and evidence-based.

## 🧠 Key Concepts Glossary

| Concept | Definition |
|---------|------------|
| **Problem-Solving Process** | A four-step model: identify problem, analyze causes, develop solutions, implement |
| **Logic Tree** | A visual diagram breaking a problem into MECE sub-components |
| **Yes/No Tree** | A decision tree using binary (yes/no) questions to narrow options |
| **Hypothesis Pyramid** | A hierarchical structure of hypotheses, from the main hypothesis to sub-hypotheses |
| **Root Cause** | The fundamental underlying cause of a problem, not just its symptoms |
| **Design Plan** | A structured plan for problem-solving analysis, including hypotheses and data needed |
| **Pros and Cons** | Listing advantages and disadvantages of each option |
| **Criteria and Evaluation** | Using defined criteria to objectively evaluate options |
| **MECE** | Mutually Exclusive, Collectively Exhaustive — the principle of complete, non-overlapping categories |
| **80/20 Rule** | Focusing on the few factors that drive the majority of the impact |
